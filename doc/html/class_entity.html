Detailed Description

Any Object that is affected by gravity or capable of random motion.

Contrary to popular definition, an Entity is defined in S2M as any type of object (living, mechanic, inanimated) that is capable of being affected by gravity and having random motion at any given time of the game.

void Entity::update ( )
virtual

Overridable update method.

If you derive the Entity class be sure to place

Entity::update();

on the END of the new class' update method so it can implement the Entity default update method, which implements Object's as well.

Default Sprite facing is left, so change this is it's not your case.

When facing left, facing is equal to false.

void Entity::updateSpr ( )
virtual

Overridable method. Updates all sprite-related variables.

Internally, it is always called after the general update method. This is because we don't want to update the Sprite until we've done the logic of the Object. Think about it as a simple Input->Output procedure.
